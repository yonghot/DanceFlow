# DanceFlow PRD 분석 — 기능 분류 및 AC

> PRD.md 기반 우선순위 분류 + 인수 조건 (Acceptance Criteria) + 의존 관계

## P0 — MVP 필수 (구현 완료)

### F1. 실시간 포즈 감지
- **AC1**: MediaPipe BlazePose 33개 랜드마크를 30fps 이상으로 추적
- **AC2**: GPU 가속 사용, 불가 시 CPU 폴백
- **AC3**: 카메라 권한 거부/미발견/사용중 에러를 한국어 안내로 표시
- **AC4**: 스켈레톤 오버레이가 미러링된 카메라 위에 실시간 렌더링
- **의존**: 없음 (독립 모듈)
- **상태**: ✅ 완료

### F2. 정확도 스코어링
- **AC1**: 레퍼런스 비교 시 정확도(70%) + 타이밍 DTW(30%) 가중 평균
- **AC2**: 레퍼런스 없을 시 트래킹 품질 점수 (visibility 기반)
- **AC3**: 5단계 판정 (Perfect 95+, Great 80-94, Good 65-79, OK 50-64, Miss 0-49)
- **AC4**: 스코어링 결과 100ms 이내 계산
- **의존**: F1 (포즈 데이터), F5 (레퍼런스 데이터)
- **상태**: ✅ 완료

### F3. 신체부위별 분석
- **AC1**: 5개 구역 (좌팔, 우팔, 상체, 좌다리, 우다리) 개별 점수
- **AC2**: 취약 부위 Progress bar로 시각화
- **의존**: F2 (스코어링 결과)
- **상태**: ✅ 완료

### F4. 인증 시스템
- **AC1**: 이메일/비밀번호 회원가입 + 로그인
- **AC2**: Google OAuth 소셜 로그인
- **AC3**: 보호 라우트 미인증 시 /login 리다이렉트
- **AC4**: 인증 사용자가 /login 접근 시 / 리다이렉트
- **AC5**: 자동 프로필 생성 (DB 트리거)
- **AC6**: Supabase 미설정 시 크래시 없이 동작 (graceful degradation)
- **의존**: Supabase 프로젝트 설정
- **상태**: ✅ 완료

### F5. 레퍼런스 녹화
- **AC1**: 카메라로 30초 레퍼런스 녹화
- **AC2**: CompactPoseData 변환 + Gzip 압축
- **AC3**: Supabase Storage 업로드 + reference_poses 레코드 생성
- **AC4**: 안무별 Primary 레퍼런스 관리
- **AC5**: 녹화 후 프리뷰 (Canvas 재생)
- **의존**: F1 (포즈 감지), F4 (인증)
- **상태**: ✅ 완료

### F9. 게임 이펙트
- **AC1**: 실시간 판정 팝업 (PERFECT/GREAT/GOOD, 0.5초 쿨다운)
- **AC2**: 콤보 카운터 (3+ 표시, 10+ 그라데이션)
- **AC3**: 스켈레톤 글로우 (품질 기반 동적 조절)
- **AC4**: 화면 플래시 (PERFECT/GREAT 판정 시)
- **AC5**: 프로그레스 바 (gradient-primary)
- **의존**: F1 (포즈 품질 데이터)
- **상태**: ✅ 완료

### F10. 결과 화면
- **AC1**: 종합 점수 카운트업 애니메이션 (1.5초)
- **AC2**: 등급 배지 + 글로우 효과
- **AC3**: 부위별 분석 Progress bar
- **AC4**: Perfect/Great 파티클 배경
- **AC5**: 등급별 한글 축하 메시지
- **의존**: F2, F3
- **상태**: ✅ 완료

## P1 — 베타 이후 (미구현)

### F6. 성장 기록 관리
- **AC1**: 연습 이력 리스트 (최근 50건)
- **AC2**: 연속 연습일수 (streak) 계산
- **AC3**: 안무별 최고 점수 추적
- **AC4**: 캘린더 뷰 연습일 표시
- **AC5**: 뱃지 시스템 (첫 연습, 연속 7일 등)
- **의존**: F4 (인증), practice_sessions 데이터
- **상태**: 🔶 부분 완료 (마이페이지 통계/기록 있음, 캘린더/뱃지 미구현)

### F7. 재생 컨트롤
- **AC1**: 속도 조절 0.5x / 0.75x / 1x
- **AC2**: 미러 모드 토글
- **AC3**: 구간 반복 연습 (시작점/끝점 설정)
- **의존**: F5 (레퍼런스 데이터)
- **상태**: ❌ 미구현 (설정에 mirrorMode/playbackSpeed 필드만 존재)

### F8. SplitView 비교 화면
- **AC1**: 좌측 레퍼런스 영상, 우측 사용자 카메라
- **AC2**: 동기화된 재생
- **AC3**: 실시간 점수 오버레이
- **의존**: F5 (레퍼런스), F7 (재생 컨트롤)
- **상태**: ❌ 미구현

## 기술 부채

1. **레퍼런스 포즈 미확보**: 6곡 시드 데이터 모두 `referencePoses: []` → 실제 녹화 필요
2. **테스트 커버리지 30%**: 코어 로직 중심 확대 필요 (목표 80%+)
3. **Dexie.js 잔재**: PRD에 IndexedDB/Dexie 언급 있으나 Supabase로 전환 완료 → 문서 정리 필요
4. **오프라인 모드 미지원**: 원안의 오프라인 지원 → Supabase 전환으로 온라인 필수
